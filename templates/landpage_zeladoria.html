Sistema de Mapeamento da Limpeza de Salas

Nascido de uma demanda real do Senac CEP Mossoró para otimizar o gerenciamento da limpeza das salas, este projeto foi estruturado como uma prática pedagógica integrada para os alunos do curso Técnico em Desenvolvimento de Sistemas. Simulando um ambiente profissional, o instrutor Thiago Medeiros desenvolveu a API backend ("Sistema de Mapeamento da Limpeza de Salas"), enquanto os alunos, com orientação da instrutor, assumiram o desafio de criar individualmente o aplicativo mobile que consome essa API. O sistema permite gerenciar e mapear as salas limpas, fornecendo informações atualizadas sobre a higienização dos ambientes.

construction Atenção: O projeto ainda está em desenvolvimento ativo. Novas funcionalidades e melhorias serão adicionadas futuramente.

Proposta Pedagógica

Este projeto nasceu como uma iniciativa prática e contextualizada, alinhada ao Modelo Pedagógico Senac, que valoriza a aprendizagem por competências e o protagonismo do aluno. Orientados pelo instrutor Thiago Medeiros e com o suporte da instrutora Maria Paula, os alunos foram desafiados a aplicar seus conhecimentos na resolução de um problema real da instituição: a gestão manual e potencialmente ineficiente da limpeza das salas.

A abordagem integrou diversas Unidades Curriculares, simulando um ambiente de desenvolvimento profissional onde os alunos (desenvolvedores mobile frontend) consumiram uma API RESTful robusta (backend) fornecida e gerenciada pelo instrutor Thiago Medeiros. Isso permitiu trabalhar não apenas aspectos técnicos (React Native, TypeScript, APIs), mas também as Marcas Formativas Senac como:

  • Domínio técnico-científico: Aplicação de tecnologias atuais e boas práticas de desenvolvimento.
  • Criatividade e atitude empreendedora: Proposição de uma solução tecnológica para um problema existente.
  • Visão crítica: Análise do problema e dos requisitos para o desenvolvimento da solução.
  • Colaboração e comunicação: Interação com a API (contrato definido) e uso de versionamento (GitHub).
  • Atitude Sustentável: Otimização de processos, potencialmente reduzindo desperdícios de tempo e recursos.
  • Autonomia digital: Uso de ferramentas e tecnologias para criar a aplicação.

O projeto visa validar a capacidade dos alunos de transformar um requisito de negócio em um produto de software completo e funcional, preparando-os para os desafios do mercado de trabalho sob a orientação dos instrutores.

Exemplos de Endpoints

1. Login (Obter Token)

Use as credenciais acima para obter um token de autenticação, necessário para as demais requisições.

POST /api/accounts/login/
Content-Type: application/json

{
    "username": "zelador", // ou "colaborador"
    "password": "Senac@098"
}

Resposta de Sucesso (200 OK): Retorna o token e dados do usuário.

{
    "token": "seu_token_aqui...",
    "user_data": { ... }
}

2. Listar Salas

Retorna a lista de salas cadastradas e seus status atuais. Requer autenticação.

GET /api/salas/
Authorization: Token seu_token_aqui

Resposta de Sucesso (200 OK): Retorna um array de objetos de sala.

[
    {
        "id": 1,
        "qr_code_id": "uuid-da-sala-1",
        "nome_numero": "Laboratório de Redes",
        "status_limpeza": "Limpeza Pendente",
        "ultima_limpeza_data_hora": null,
        "detalhes_suja": null,
        ...
    },
    {
        "id": 2,
         "qr_code_id": "uuid-da-sala-2",
        "nome_numero": "Auditório Principal",
        "status_limpeza": "Suja",
        "ultima_limpeza_data_hora": "2025-10-17T10:00:00Z",
        "detalhes_suja": {
             "data_hora": "2025-10-17T15:30:00Z",
             "reportado_por": "colaborador",
             "observacoes": "Café derramado."
        },
        ...
    }
]

3. Iniciar e Concluir Limpeza (Apenas Zeladoria)

Para registrar uma limpeza, o usuário zelador deve:

  1. Iniciar a limpeza: 
    POST /api/salas/{qr_code_id}/iniciar_limpeza/
    (Retorna o ID do registro de limpeza).
  2. Adicionar pelo menos uma foto de comprovação: 
    POST /api/fotos_limpeza/
    (Requisição multipart/form-data contendo o ID do registro e o arquivo de imagem).
  3. Concluir a limpeza: 
    POST /api/salas/{qr_code_id}/concluir_limpeza/
    (Pode incluir observações no corpo JSON).

4. Marcar Sala como Suja (Apenas Solicitante)

O usuário colaborador pode reportar uma sala suja:

POST /api/salas/{qr_code_id}/marcar_como_suja/
Content-Type: application/json

{
    "observacoes": "Lixo acumulado na lixeira." // Opcional
}

Para mais detalhes sobre todos os endpoints disponíveis, filtros e estruturas de dados, consulte a documentação completa no repositório da API.
